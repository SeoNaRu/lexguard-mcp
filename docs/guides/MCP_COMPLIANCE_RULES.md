## 🚀 MCP 서버 생성 가이드

### 필수 요구사항

#### 1. MCP 스펙 버전
- **최소 지원 버전: 2025-03-26 이상**
- [MCP 스펙 문서](https://modelcontextprotocol.io/specification/2025-03-26) 준수 필수

#### 2. 전송 방식
- **Streamable HTTP 방식만 지원**
- stdio 방식은 지원하지 않음
- [MCP 전송 스펙 문서](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http) 참고

#### 3. 서버 유형
- **Remote MCP 서버만 지원**
- 공개된 URL로 접근 가능한 도메인이어야 함
- 로컬 서버는 지원하지 않음

#### 4. 서버 설계
- **Stateless MCP 서버 권장** (no session)
- 세션 상태를 유지하지 않는 무상태 서버 설계

#### 5. 인증 방식
- 사용자 인증이 필요한 경우 다음 중 하나를 지원해야 함:
  - **OAuth 인증**
  - **커스텀 헤더 방식**
- [MCP 인증 스펙 문서](https://modelcontextprotocol.io/specification/2025-03-26/basic/authorization#authorization-flow) 참고

#### 6. 사전 점검
- **MCP Inspector로 MCP 표준 스펙 준수 여부 사전 점검 필수**
- [MCP Inspector 설치](https://modelcontextprotocol.io/docs/tools/inspector)

#### 7. SDK 사용
- **활발하게 운영되는 SDK를 사용하거나 참조해야 함**
- [SDK 목록 보기](https://modelcontextprotocol.io/docs/sdk)

## ✅ 기본 심사 정책

### Tool 개수
- **최소 1개 이상의 Tool 포함 필수**
- **권장 Tool 개수: 3~20개**
- 1개의 MCP Server가 너무 많은 Tool을 보유한 경우 AI가 올바른 툴 선택을 하기 어렵습니다.

### 심사 후 관리
- 심사 완료 후에도 정책 위반이 감지되면 관리자 판단에 따라:
  - Tool이 비활성화되거나
  - 등록이 철회될 수 있음
- 이 경우 사유는 신청자에게 개별 안내됩니다.

### 반려 사유
다음과 같은 경우 심사가 반려될 수 있습니다:

1. **표준 스펙 위배**
   - 등록한 MCP가 표준 스펙에 위배되거나 비정상적인 동작이 감지되는 경우

2. **제3자 서비스 인증 정보**
   - 자사 서비스가 아닌 제3의 서비스 인증 정보를 사용하는 경우
   - 등록자와 인증 정보사의 관계에 따라 심사가 반려될 수 있음

3. **자동 생성 플랫폼**
   - MCP Server를 자동 생성하는 제3자 플랫폼(ex. https://zapier.com/mcp)에서 생성된 MCP는 등록이 허용되지 않음
   - 명확한 소유자 검증 및 정책 준수가 어려움

4. **중복 기능**
   - 동일 기능 구조의 MCP가 반복적으로 공개 MCP 등록 심사를 요청하는 경우 등록이 제한될 수 있음
   - ※ 단순히 명칭, 문구, 출력 형식만 변경한 MCP는 별도의 신규 MCP로 인정되지 않음

5. **기본 기능 중복**
   - LLM이 자체 웹 검색을 통해 충분히 구현 가능한 기능만을 제공하는 MCP는 심사 과정에서 반려 또는 공개 제한될 수 있음
   - ※ 단, LLM의 기본 기능을 확장하거나 기능을 추가하는 목적이 명확히 드러날 경우 예외적으로 허용

6. **품질 기준 미달**
   - 내부 MCP 품질 기준(안정성, 창의성, 응답 일관성 등)에 미달하는 경우
   - 관리자 판단에 따라 비공개 또는 개선을 요청할 수 있음

7. **명칭 및 설명**
   - MCP의 이름과 설명은 기능을 명확히 전달해야 함
   - 추상적이거나 모호한 표현이 포함된 경우 심사 중 수정 요청이 이루어질 수 있음

8. **유료 결제 필수**
   - 등록한 MCP의 도구 실행에 유료 결제 계정 또는 유료 구독이 필수적으로 요구되는 경우 반려 사유가 될 수 있음

## 🔧 Tool의 동작

### 필수 사항

1. **사전 테스트 필수**
   - 등록된 모든 Tool은 실제 동작 여부를 반드시 사전 테스트 해야 함
   - 오류 없이 응답이 오는지 확인해야 함

2. **성능 요구사항**
   - 응답이 지나치게 느리거나 timeout이 자주 발생하는 경우 반려 사유가 될 수 있음
   - Tool이 정상 동작하더라도, 과도한 리디렉션, 크롤링 지연, 불필요한 외부 호출이 감지될 경우 성능 저해로 반려될 수 있음

3. **응답 크기 제한**
   - **Tool의 Response가 24k를 초과하면 에러로 처리되기 때문에 반려 사유가 될 수 있음**
   - Response 텍스트가 과도하게 크지 않도록 확인 필요

### 허용 사항

- 필요한 경우 Tool의 답변에 아웃 링크가 포함되어도 좋습니다.

### 금지 사항

다음과 같은 경우 반려 사유가 될 수 있습니다:

1. **상업적 행위**
   - 답변 내 상업적 링크, 구매 유도, 리워드 제공 등 명시적 또는 암묵적 상업 행위가 지나치게 포함될 경우

2. **부적절한 내용**
   - 답변에 비속어/정치적/성적인 내용이 포함되어 있거나 일반적인 사회 통념에 위배되는 경우

3. **보안 위험**
   - 응답에 보안상 위험이 있거나 악의적으로 이용될 수 있는 파일 다운로드를 포함하거나 유도할 경우
   - 심사가 반려되거나 MCP가 비공개 처리될 수 있음

## 📝 Name / Description 규칙

### Tool Name 규칙

1. **"kakao" 사용 제한**
   - MCP Server Name 또는 Tool Name에 "kakao"를 prefix 또는 suffix로 사용할 수 없음
   - 꼭 필요한 경우 사전 협의가 필요함

2. **명칭 규칙**
   - Name은 간결하게, 서비스 기능을 대표할 수 있는 명칭이어야 함
   - 'AI', 'Bot', 'Service' 등의 중복 키워드는 지양

### Description 규칙

1. **상세한 기술**
   - Tool Description은 LLM이 잘 이해할 수 있도록 자세하게 기술되어야 함

2. **언어**
   - 언어는 영어 또는 한국어를 권장

## 🔒 개인정보

### 수집 제한

- Tool의 기능과 무관한 개인정보를 수집하거나 요구해서는 안됩니다.

### 전송 제한

- 반드시 필요한 경우를 제외하고는 사용자의 인증 정보나 민감 정보를 외부로 전송하여서는 안됩니다.

### 인증 정보 사용

- OAuth 인증이나 Token / Key 입력 등을 통해 획득한 인증 정보는 Tool의 사용 목적 외에 다른 용도로 사용해서는 안됩니다.

### 금지되는 정보

Tool이 아래 정보를 요구하거나 response로 전송하는 경우 심사가 반려될 수 있습니다:

- 주민등록번호
- 운전면허번호
- 여권번호
- 외국인 등록번호
- 카드번호
- 계좌번호

## 📬 심사 진행 및 상태 안내

### 심사 기간

- 심사 요청한 MCP는 **영업일 기준 7일 이내**에 심사가 완료됩니다 (평균 1~2일)

### 안내 메일

- 심사 진행 상태에 대한 안내 메일은 **"내 정보 관리 > 연결된 이메일"**의 이메일 주소로 보내집니다.
- 심사 중 별도의 문의 또는 보완 요청이 발생할 수 있으며, 동일한 이메일로 메일이 발송됩니다.

### 문의

- 심사에 관한 추가 문의사항은 **PlayMCP 디스코드**를 이용해 주세요.

### 정보 갱신

- 심사가 완료되어 게시 중인 MCP의 Tool 정보가 변경될 경우 반드시 **MCP 정보 수정에서 "정보 불러오기"를 다시 시행**하여 주셔야 정보가 갱신됩니다.
- 이 때 재심사가 진행됩니다.

---

## 📌 기타 사항

### 인증 처리

- 인증이 필요한 상황에 인증 정보가 없거나 만료된 경우 **401(unauthorized) Http 응답이 필요**합니다.

### 서버 안정성

- 심사가 통과된 공개 MCP의 서버가 불안정하거나 지속적으로 응답이 없는 경우, 운영자가 MCP를 비활성화할 수 있습니다.

### 대표 이미지

- MCP 대표 이미지에 움직이는 이미지는 사용할 수 없습니다.
- MCP의 대표 이미지는 서비스의 성격에 부합해야 하며, 저품질 또는 부적절한 이미지를 사용할 경우 심사에서 반려될 수 있습니다.

### Resource와 Prompt

- PlayMCP는 현재 MCP 스펙의 Resource와 Prompt 정보는 다루지 않습니다.

## ✅ 체크리스트

MCP 서버를 제출하기 전에 다음 사항을 확인하세요:

### 기술적 요구사항
- [ ] MCP 스펙 버전 2025-03-26 이상 준수
- [ ] Streamable HTTP 방식으로 구현
- [ ] Remote 서버로 공개 URL 제공
- [ ] Stateless 서버 설계 (no session)
- [ ] OAuth 또는 커스텀 헤더 인증 지원 (필요한 경우)
- [ ] MCP Inspector로 사전 점검 완료
- [ ] 활발하게 운영되는 SDK 사용

### Tool 요구사항
- [ ] 최소 1개 이상의 Tool 포함
- [ ] Tool 개수 3~20개 권장 범위 내
- [ ] 모든 Tool 사전 테스트 완료
- [ ] 응답 속도 및 안정성 확인
- [ ] Response 크기 24k 이하 확인

### 정책 준수
- [ ] Tool Description이 명확하고 상세함
- [ ] Name이 간결하고 기능을 대표함
- [ ] "kakao" prefix/suffix 미사용 (필요시 사전 협의)
- [ ] 개인정보 수집/전송 최소화
- [ ] 금지된 개인정보 요구/전송 없음
- [ ] 상업적 행위 없음
- [ ] 부적절한 내용 없음
- [ ] 보안 위험 요소 없음

### 기타
- [ ] 인증 실패 시 401 응답 구현
- [ ] 대표 이미지 준비 (움직이는 이미지 아님)
- [ ] 서버 안정성 확보
